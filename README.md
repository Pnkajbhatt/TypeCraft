# TypeCraft
> **Craft your typing. Master your profession.**

TypeCraft is an AI-powered typing practice platform that goes beyond traditional tools. Instead of generic predefined text, TypeCraft generates **profession-specific paragraphs** tailored to your field — so every keystroke makes you faster at the language you actually use every day.

---

## The Problem

Every typing tool out there gives you the same sentences — random quotes, generic paragraphs, predefined templates. A doctor, a lawyer, and a software engineer all get the same text to practice on.

**That's not how professionals work.**

---

## The Solution

TypeCraft uses AI to generate dynamic, profession-aware paragraphs. A doctor practices with medical terminology. A lawyer practices with legal language. A coder practices with technical documentation. Every session feels natural, relevant, and purposeful.

---

## Features

- **Guest Practice** — Jump straight into typing with no signup required (predefined paragraphs)
- **Profession-Based Practice** — Login and select your profession for AI-generated, relevant paragraphs
- **Never Repeat** — Smart tracking ensures you never see the same paragraph twice
- **Live Metrics** — Real-time WPM, accuracy, and mistake tracking during every session
- **Progress Dashboard** — Visual graphs and session history to track your improvement over time
- **Dynamic Pool System** — AI silently refills your paragraph pool in the background, so you never wait

---

## Tech Stack

### Frontend
- **React.js** — Component-based UI
- **Custom Hooks** — `useTyping` hook handles all typing logic (WPM, accuracy, timer, mistakes)
- **Recharts** — Progress graphs on profile dashboard
- **Axios** — Centralized API service layer

### Backend
- **Node.js + Express.js** — RESTful API server
- **PostgreSQL** — Relational database
- **JWT** — Secure authentication
- **node-cron + Bull Queue** — Background jobs for AI paragraph pool management

### AI
- **Anthropic Claude API** — Profession-aware paragraph generation

## Database Schema

```
Users
 └── profession_id → Professions

Professions
 └── id, name

Paragraphs
 └── profession_id → Professions
 └── text, is_used, created_at

Progress
 └── user_id → Users
 └── paragraph_id → Paragraphs
 └── wpm, accuracy, mistakes, time_taken

User_Paragraphs (Junction)
 └── user_id → Users
 └── paragraph_id → Paragraphs
 └── seen_at
```

---

## API Routes

```
POST   /api/auth/register       — Register new user
POST   /api/auth/login          — Login and receive JWT token
GET    /api/paragraphs/next     — Fetch next unseen paragraph (auth required)
POST   /api/sessions/complete   — Save completed session results (auth required)
GET    /api/users/progress      — Fetch user progress and history (auth required)
```

---

## How the Paragraph Pool Works

1. Each profession has a **shared pool of 40 paragraphs** generated by AI
2. Every user gets tracked individually — **no paragraph repeats per user**
3. When a pool reaches **80% usage**, a background job silently generates new paragraphs
4. Paragraph **text refreshes**, but IDs are preserved — so progress history stays intact
5. Users experience **zero waiting** — the next paragraph is always ready

## Getting Started

### Prerequisites
- Node.js v18+
- PostgreSQL v14+
- Anthropic API Key

### Installation

```bash
# Clone the repository
git clone https://github.com/yourusername/typecraft.git
cd typecraft

# Install server dependencies
cd server && npm install

# Install client dependencies
cd ../client && npm install
```

### Environment Variables

Create a `.env` file in the `server/` directory:

```env
PORT=5000
DATABASE_URL=postgresql://localhost:5432/typecraft
JWT_SECRET=your_jwt_secret
ANTHROPIC_API_KEY=your_anthropic_api_key
```

### Run the Project

```bash
# Start backend
cd server && npm run dev

# Start frontend
cd client && npm run dev
```
